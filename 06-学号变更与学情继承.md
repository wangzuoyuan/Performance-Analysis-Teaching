# 06 · 学号变更与学情继承（分班场景）

> **本章修订 01·假设 A1。** 原假设「`student_id` 跨学年稳定」**不成立**：
> 高一→高二/高三经历分班，**学号会变**，教学班数量与名称也变。
> 现有应用所有学生画像 / 跨学年趋势都按单一 `student_id` 聚合，会因此断裂——必须新增「学生身份（Person）」层。

## 1. 问题与目标

| 场景 | 期望行为 |
|------|----------|
| 高一学生进入我高二/三的新教学班，是同一人（但新学号） | **学情继承**：学生画像、跨学年趋势、谈话/作业档案自动接续高一记录 |
| 新教学班里来的是我没教过的新学生 | **从当下开始记录**；并提供「可选导入他高一成绩」入口 |
| 学号变了，但人没变 | 系统能把「高一学号」与「高二学号」认成同一人 |

核心：把「**学号(per 学段)**」与「**人(跨学段)**」解耦。分析按需在两层间解析。

## 2. 跨学年身份的锚点：没有现成稳定唯一键，以人工确认为主

> **实测纠正**：源 `SubjectScore.xueji` 字段**不是**学籍号身份——库里只有 `1/3/4` 三个值、
> 全体学生共用（范围 1–4），它是**学籍状态/类别码**（在籍/借读之类），无法认人。
> `student_id`（如 `7250601`）就是会随分班变化的校内学号。
> **结论：当前数据里不存在可自动锚定「跨学年同一人」的稳定唯一字段。**

因此跨学年身份链接采用「**人工确认为主 + 对照表批量为辅**」，不做按字段自动并档：

- **主路径 · 姓名匹配 + 人工确认**：老师配高二/三教学班、加成员时，系统按姓名去高一记录里找候选，
  老师逐个确认「是我高一的某某 / 是新学生」。候选项附**高一行政班 + 最近主三门名次/学号**辅助辨认同名。
- **辅路径 · 学号对照表导入**：若学校能提供「高一学号 ↔ 高二学号」对照表（Excel/CSV），
  一次性导入批量建立链接，省去逐个确认。
- **绝不**按姓名自动链接（同名会错并档案）；一切链接要么来自老师确认，要么来自对照表。

> 若后续发现本校另有真正稳定唯一的标识（如身份证号、全国学籍号 G 开头那串），
> 可作为可选自动锚点接入 `student_identity.ext_key`（见 §3.1），但**默认不依赖它**。

## 3. 数据模型新增

### 3.1 `student_identity` — 人（跨学段身份）
```python
class StudentIdentity(Base):
    __tablename__ = "student_identity"
    id = Column(Integer, primary_key=True)
    display_name = Column(String, nullable=True)
    ext_key = Column(String, nullable=True, index=True)  # 预留：身份证/全国学籍号等真稳定唯一键（默认不用）
    gender = Column(String, nullable=True)
    note = Column(String, nullable=True)
    created_at = Column(DateTime, default=datetime.utcnow)
```
> 注意：**不要**用 `SubjectScore.xueji` 当身份键——实测它是 1/3/4 的状态码，非唯一。

### 3.2 `student_alias` — 学号 ↔ 人
```python
class StudentAlias(Base):
    __tablename__ = "student_alias"
    id = Column(Integer, primary_key=True)
    identity_id = Column(Integer, ForeignKey("student_identity.id"), nullable=False)
    student_id = Column(String, nullable=False)          # 某学段的学号
    grade = Column(Integer, nullable=True)               # 该学号所属年级（1/2/3）
    link_source = Column(String, nullable=False, default="name_confirmed")  # name_confirmed / crosswalk / manual / ext_key
    created_at = Column(DateTime, default=datetime.utcnow)
    __table_args__ = (
        UniqueConstraint("student_id", name="uq_alias_student"),  # 一个学号只属一个人
        Index("idx_alias_identity", "identity_id"),
    )
```
- 一人多学号（高一一个、高二一个…）；一个学号唯一属于一个人。
- **未链接的学号**：不必强制每个学号都建 identity——读侧解析时「查不到 alias 就把该学号当作独立单人」，保证零配置也能用（无分班/未确认时退化为现状行为）。

**应用位置（仅“以学生为中心、需要跨学年连续”的读侧才解析人）：**

| 读侧 | 改法 |
|------|------|
| `GET /students/{id}`（画像/跨学年趋势/班级排名） | 先 `ids = student_ids_of_person(student_id)`，所有 `TotalScore/SubjectScore` 查询用 `student_id IN ids`；趋势点按当年学号→当年教学班标注 |
| 谈话/档案 `notes`、作业 `homework` 的学生页读取 | 同样按 `ids` 并集查，使高一档案/作业在高二画像里可见（继承学情） |
| 学生检索 `/api/students` | 列表项以「人」去重（同一人多学号合并为一行，显示当前年级学号），避免一人出现两次 |
| 重点关注 / 段位 / 对比等**单场考试/单年级**分析 | **不解析人**：年级内学号本就稳定，保持按 `student_id`，零成本 |

> 设计要点：身份解析是**叠加层**，只在跨学年学生视图启用；单年分析不动，回归风险小。

## 5. 教学班成员的「继承 / 新建 / 导入」流程

配置高二/三教学班、加成员时（扩 03·§2 的 `members/import` 与 `teaching/service`）：

对每个待加入学生（来自花名册学号或姓名清单），按下列顺序判定：

1. **已有对照表链接** → 直接 `继承`（链接已由 §2 辅路径建立）。
2. **姓名命中高一记录** → 返回候选，老师确认：
   - 确认是同一人 → 建 alias(`link_source=name_confirmed`)，`继承`；
   - 确认是新学生 → 不链接，`新建`。
   - 同名多人 → `ambiguous`，列候选（高一行政班/最近名次/学号）让老师选。
3. **零命中** → `新建`：按当下开始记录；额外提供两种「导入高一成绩」选项：
   - (a) **关联已有高一学号**：老师手动指定其高一学号 → 建 alias(`link_source=manual`)，继承；
   - (b) **上传该生高一成绩 Excel**：作为该 identity 的历史数据入库（打 grade=1 标记），趋势可见。

成员导入接口返回三态升级为四态：`inherited`（已继承）/ `ambiguous`（同名待确认）/ `new`（新学生）/ `unmatched`（无学号待补）。

## 6. 前端（扩 04·§1 配置向导 + §6 学生画像）

- **配置向导成员表**：每个成员显示来源徽章——`继承(学籍)` / `继承(已确认)` / `新` ；`新`行附「关联高一学号 / 导入高一成绩」操作。同名候选用 Dialog 让老师选「是同一人 / 是新学生」，候选项展示高一行政班 + 最近主三门名次帮助辨认。
- **学生画像页**：头部显示「学段履历」——高一(学号A·X班) → 高二(学号B·物A1)；趋势图跨学段连贯，断点用不同年级底色；若该生为「新建未继承」，趋势仅从高二起，并显示「导入高一成绩」入口。
- **学生检索**：同一人合并一行，班级列显示当前教学班，可展开看历史学号。

## 8. 执行步骤增补（插入 05 的对应阶段）

- **Phase 0**：确认本校有无真稳定唯一标识（身份证/全国学籍号）；有则接 `ext_key`，没有则按本章「人工确认+对照表」推进。**不要把 `xueji` 当身份键**。
- **Phase 1**：建 `student_identity`/`student_alias`（迁移**只建表，不自动并档**）。
- **Phase 2**：`scope.py` 加 `identity_of` / `student_ids_of_person` / `name_candidates` / `link_aliases` / `import_crosswalk`；`test_identity.py`（姓名候选、确认建链、对照表导入、未链接退化为单人、跨学号并集）。
- **Phase 3**：（无自动链接钩子）上传保持原状；身份链接由配置流程触发。
- **Phase 4**：`students/{id}` 与 notes/homework 学生读改身份并集；`/api/students` 按人去重。
- **Phase 6/7**：配置向导四态 + 继承/导入交互；学生画像学段履历；检索去重。
- 工期：在原 13–15 人日基础 **+约 2.5–3 人日**。

## 9. 边界与口径

- **同名多人**：必须人工确认，绝不自动按姓名链接（避免错并档案）。
- **错链可撤销**：提供「解除关联」（删 alias），把学号还原为独立人。
- **作业/档案归属**：继承后，高一档案在高二画像可见（按身份并集读）；但作业缺交看板等**单年统计**仍按当年学号、当年教学班，不跨年混算。
- **段位/年级排名**：始终单年级口径，不受身份层影响。
- **导入的高一成绩**：仅用于该生历史趋势展示，不参与高二年级排名/班均分计算（避免污染当年全年级口径）。
