# 02 · 数据模型蓝图

源：`backend/app/db/models.py`。SQLite + SQLAlchemy，`Base.metadata.create_all` 启动即建表（无 Alembic）。
**新增表会自动建；改既有表列需一次性迁移脚本**（见 §4）。

## 1. 新增表

### 1.1 `teaching_class` — 老师的教学班
```python
class TeachingClass(Base):
    __tablename__ = "teaching_class"
    id = Column(Integer, primary_key=True)
    grade = Column(Integer, nullable=False)          # 1/2/3
    label = Column(String, nullable=False)           # "1" / "6" / "物A1" / "史B3"
    subject = Column(String, nullable=True)          # 任教学科，如 "物理"；走班单科分析用
    kind = Column(String, nullable=False, default="教学")  # "行政"(高一) / "教学"(走班)
    note = Column(String, nullable=True)
    sort_order = Column(Integer, nullable=False, default=0)  # 老师自定义展示顺序
    created_at = Column(DateTime, default=datetime.utcnow)
    __table_args__ = (UniqueConstraint("grade", "label", name="uq_tc_grade_label"),)
```
- 唯一键 `(grade, label)`：同年级同名只一条。跨年级允许重名（高一(1)班 与 高二(1)班 是两条）。
- `kind`：高一配置出来的标记 `行政`（成员可由 `class_num` 自动推导）；高二/三标记 `教学`。

### 1.2 `teaching_class_member` — 成员映射（教学班 ↔ 学号）
```python
class TeachingClassMember(Base):
    __tablename__ = "teaching_class_member"
    id = Column(Integer, primary_key=True)
    teaching_class_id = Column(Integer, ForeignKey("teaching_class.id"), nullable=False)
    student_id = Column(String, nullable=False)      # 真实学号
    source = Column(String, nullable=False, default="manual")  # parser / manual / roster / class_num
    created_at = Column(DateTime, default=datetime.utcnow)
    __table_args__ = (
        UniqueConstraint("teaching_class_id", "student_id", name="uq_tcm_class_student"),
        Index("idx_tcm_student", "student_id"),
        Index("idx_tcm_class", "teaching_class_id"),
    )
```
- `source` 记录成员怎么来的，便于「自动同步」时只覆盖自己来源的记录、保留手工增删。
- 一个学生可属于同年级多个教学班吗？现实里走班一个学生一科一班，但不同老师视角下可能多班——本工程是**单老师**视角，同一老师的同年级一个学生通常只在一个教学班；不强制全局唯一，仅 `(class, student)` 唯一。

## 2. 改既有表

### 2.1 `subject_score` 增列 `class_label`
```python
class_label = Column(String, nullable=True)   # 教学班标签；解析器填，缺省 None
Index("idx_subject_class_label", "exam_id", "class_label")
```
- 解析器有教学班列时写入；没有则 `None`。读侧成员解析优先用成员表，不直接依赖此列，但它支撑「自动建班」与回退。

### 2.2 `class_average` 增列 `class_label`
```python
class_label = Column(String, nullable=True)   # 班均分行对应的标签
```
- 解析器：班均分表带教学班名时写 `class_label`；只有行政班数字时 `class_label = str(class_num)`。
- 对比页统一用 `class_label` 做横轴键；为空时回退 `str(class_num)`。

### 2.3 `teacher` 调整
- **废弃**单班字段 `target_class_high1/2/3`（保留列以兼容旧库，但不再写、不再读）。
- 新增可空 `current_teaching_class_id = Column(Integer, nullable=True)`：记住上次选中的「当前班」，用于默认视图与侧栏展示。
- `name` / `school` 不变。

### 2.4 复用 `class_roster`（作业模块花名册）
- 已有列：`student_id`(PK,真实学号)、`name`、`class_num`、`seat_no`、`gender`、`excluded`。
- 手工花名册导入时，可同时写 `class_roster`（补 name/座号）与 `teaching_class_member`（建成员关系）。两者职责区分：
  - `class_roster`：作业模块的学生主体（含座号/排除），**按行政班或单一班**；
  - `teaching_class_member`：分析维度的成员关系，**按教学班**。
- 增列建议 `class_roster.class_label`（可空），让作业模块也能按教学班归类（非必须，Phase 5 选做）。

## 3. 成员关系的三条来源（对应 D1）

| source 值 | 触发时机 | 行为 |
|-----------|----------|------|
| `class_num` | 高一配置 / 上传后自动同步 | 教学班 `kind=行政` 时，成员 = 该 grade 下 `SubjectScore.class_num == int(label)` 的 distinct 学号 |
| `parser` | 上传带教学班列的成绩表 | 对 `SubjectScore.class_label == label` 的 distinct 学号，自动加入对应教学班 |
| `manual` / `roster` | 老师在配置页粘贴清单 / 上传花名册 Excel | 姓名→学号反查（见 §5），写成员 |

**自动同步原则**：上传新考试后，对 `kind=行政` 的班按 `class_num` 重算成员（覆盖 `source=class_num` 的行）；对带 `class_label` 的数据补 `source=parser` 成员。`source=manual/roster` 的成员永不被自动覆盖。

## 4. 迁移脚本（一次性，幂等）

新建 `backend/app/db/migrate_teaching.py`，参考既有 `homework/migrate.py` 风格：

1. `Base.metadata.create_all()` 建新表（`teaching_class` / `teaching_class_member`）。
2. 用 `ALTER TABLE` 给 `subject_score` / `class_average` / `class_roster` 加 `class_label` 列（SQLite 支持 `ADD COLUMN`；先 `PRAGMA table_info` 判断是否已存在，保证幂等）。
3. 给 `teacher` 加 `current_teaching_class_id` 列。
4. **数据回填**：把旧 `teacher.target_class_highN` 转成 `teaching_class` 行 + `class_num` 来源成员，让老库平滑升级（旧用户的本班变成教学版的一个班）。
5. 回填 `subject_score.class_label = NULL`（默认即可）、`class_average.class_label = str(class_num)`。

> 在 `run.py init` / 启动路径里调用一次该迁移（判断版本标记，避免重复 ALTER 报错）。可在 `homework_setting` 里存 `schema_version` 键。

## 5. 姓名 → 学号 反查（手工导入用）

- 反查源：`SubjectScore.name`（已入库成绩里的姓名）+ `class_roster.name`。
- 流程：导入清单按姓名匹配 → 唯一命中直接落学号；同名多个 → 返回待人工消歧列表（带行政班/最近考试辅助区分）；零命中 → 进「未匹配」列表让老师补学号。
- 复用 `homework/migrate.py` 已验证的「按姓名映射座号到真实学号」思路。

## 6. 表关系总图

```
teacher ──current_teaching_class_id──▶ teaching_class
teaching_class ──1:N──▶ teaching_class_member ──student_id──▶ (SubjectScore/TotalScore by student_id)
subject_score(+class_label)  ─┐
class_average(+class_label)   ─┴─▶ 解析器写入，支撑自动建班 & 对比横轴
class_roster(+class_label?)   ───▶ 作业模块主体（复用于花名册导入）
```
