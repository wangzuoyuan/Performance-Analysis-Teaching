# 01 · 需求与设计决策

## 1. 原始需求拆解

| # | 原始诉求 | 落地含义 |
|---|----------|----------|
| 1 | 移植为「教学班」成绩分析应用 | 从「一个老师一个行政班」改为「一个老师多个教学班」的数据与界面模型 |
| 2 | 初始化后先配置班级；高一=行政班，高二/三=教学班（数字或 `物A1`/`史B3`）；对比页标注我教的所有教学班 | ① 新增「班级配置」流程；② 班级标识改为**字符串**；③ 老师有**多个**班；④ 对比页高亮多班 |
| 3 | 总览仪表盘 + 分班仪表盘；总览上学生旁标注班级 | 仪表盘拆两层；学生卡/名单带教学班标签 |
| 4 | 所有质量分析分班级展示；学生检索支持全局检索 + 分班查看 | 所有分析端点支持 `teaching_class_id` 维度；学生页加全局/分班切换 |
| 5 | 只出 plan 和执行蓝图，输出到指定目录 | 本目录内的文档，不动源码 |

## 2. 已确认决策（用户拍板）

### D1 · 教学班成员来源：两者都支持
- **解析器来源**：若成绩明细表 / 班均分表带有「教学班 / 走班班级」列，解析器自动读取，写入 `class_label`，并按老师已配置的班自动建/刷成员。
- **手工来源**：成绩表只有行政班号时，老师在「班级配置」页手工建教学班并指定成员——
  - 高一：选行政班号 → 成员 = 该年级该 `class_num` 的全部学生（可随上传自动同步）；
  - 高二/三走班：粘贴学号/姓名清单，或上传教学班花名册 Excel；姓名按已入库成绩做学号反查。

### D2 · 对比页范围：全年级所有班，高亮我的
- 对比维度（横轴对象）按数据可得性决定：
  - **高一**：维度 = 行政班（数字）。展示全年级所有行政班，高亮我教的几个。
  - **高二/三**：维度 = **教学班标签 `class_label`**（成绩/班均分带教学班列时为走班名；否则回退为行政班号的字符串）。展示全年级所有班，高亮 `class_label ∈ 我的教学班集合` 的班。
- 教学班若无官方班均分行（走班无对应行政班均分），其均分由**成员学生总分现算**（源码 `compare_classes` 已有「无 ClassAverage 时按学生聚合」的回退路径，直接复用并按成员集合聚合）。

### D3 · 班级标识：统一字符串标签
- 全系统引入 `class_label: str`，对数字班（`"1"`、`"6"`）和走班（`"物A1"`、`"史B3"`）一视同仁。
- 保留原 `class_num: int` 仅作「行政班号」用途（解析自 Excel 的数字列、高一成员推导、回退展示）。`class_label` 是新的对外口径键。

### D4 · 仪表盘两层 + 全面分班
- **总览仪表盘**：聚合「我教的所有班」，每个学生/名单项旁标注其教学班标签。
- **分班仪表盘**：选定单个教学班后的专属视图（KPI、关注名单、段位分布均限定该班成员）。
- 学生检索：默认全局（我教的所有班并集），可按教学班过滤；名单带班级列。

## 3. 核心概念模型

```
Teacher (1)
   └── TeachingClass (N)          老师配置的教学班：grade + label(+subject)
            └── members: set[student_id]   成员学号集合（来源：解析器列 / 手工 / 行政班推导）

成绩数据（不变的事实层）：
   SubjectScore / TotalScore  按 student_id + exam_id 存
   每条 SubjectScore 带 class_num(行政班号) 和【新增】class_label(教学班标签，可空)

分析口径（改造层）：
   旧： WHERE SubjectScore.class_num == N
   新： WHERE TotalScore.student_id IN members_of(teaching_class_id)
```

**关键转译**：现有所有 `class_num: Optional[int]` 过滤参数 → 改为 `teaching_class_id: Optional[int]`，
后端用统一的 `resolve_scope()` 把它解析成 `set[student_id]`（`None` = 全年级），再过滤。

## 4. 关键假设与待验证项（执行时确认）

| 假设 | 说明 | 验证方式 |
|------|------|----------|
| ~~A1~~ | ~~`student_id` 跨学年稳定~~ → **已废止**：分班后学号会变 | 见 [06-学号变更与学情继承.md](06-学号变更与学情继承.md)：靠「学籍号优先 + 姓名确认回退」的身份层接续跨学年学情，不再依赖学号稳定 |
| A2 | 高二/三成绩表的班级列**可能**是行政班数字，也**可能**含教学班名 | 执行 Phase 1 时拿真实样表确认列结构；解析器需两条分支都覆盖 |
| A3 | 走班教学班无官方班均分行 | 用成员现算均分兜底（D2） |
| A4 | 老师任教班集合在一个学年内相对稳定，但跨学年会变（高一行政班→高二走班） | `teaching_class` 按 `grade` 维度区分，配置向导按年级分别配 |
| A5 | 段位阈值（高分/临界/薄弱）仍是全年级排名口径 | 不变；段位计数在「我的班成员」子集上统计，但排名仍用全年级 `xueji_rank` |

## 5. 不改动的部分（保持现状，降低风险）

- 上传两段式流程（preview/commit）、文件名解析、考试建档逻辑。
- 跨学年趋势的指标选择规则（主三门/语数英原始分；总分用 `xueji_rank`；+3 用等级分等）——见源 `chat/session.py` 业务口径，照搬。
- 作业模块、档案模块、备份模块的内部逻辑（但作业的 `class_roster` 会被「班级配置」复用，见 02）。
- 段位阈值配置 `analysis_config`（单行）。
- 部署/鉴权/数据目录机制。
