# 03 · 后端改造蓝图

源：`backend/app/`。FastAPI 路由挂 `/api`。改造遵循「读侧从 `class_num==` 切到成员集合过滤」。

## 1. 新增核心模块 `analysis/scope.py`

统一的「范围解析」，所有分析端点共用：

```python
# 伪代码
def members_of(db, teaching_class_id) -> set[str]:
    rows = db.query(TeachingClassMember.student_id)\
             .filter(TeachingClassMember.teaching_class_id == teaching_class_id).all()
    return {r[0] for r in rows}

def resolve_scope(db, *, teaching_class_id=None) -> set[str] | None:
    """返回学号集合；None 表示不限定（全年级）。"""
    if teaching_class_id is None:
        return None
    return members_of(db, teaching_class_id)

def my_class_labels(db, grade) -> dict[str, int]:
    """{label: teaching_class_id}，供对比页高亮 + 前端下拉。"""
    rows = db.query(TeachingClass).filter(TeachingClass.grade == grade).all()
    return {tc.label: tc.id for tc in rows}

def all_my_member_ids(db, grade=None) -> set[str]:
    """我教的所有班（可限定年级）的成员并集，供总览仪表盘 & 全局学生检索。"""
```

> 端点内把旧的「先查 `class_num==N` 拿学号集合」全部替换为 `resolve_scope(db, teaching_class_id=...)`。
> 旧逻辑里多处已经是「先取本班学生集合 `allowed`，再过滤 totals」（如 `band-trend`、`focus-list`），改造成本低。

## 3. 逐端点改造（`analysis/router.py`）

> 为平滑过渡可短期同时接受 `class_num`（直接当 `class_label=str(class_num)` 临时解析），但前端一律传 `teaching_class_id`。

## 4. `/class/compare` 专章（对应 D2：全年级所有班，高亮我的）

返回结构升级为按 `class_label` 组织，并标 `mine`：

```jsonc
{
  "exams": [{
    "exam_id": 1, "exam_name": "...", "grade": 2,
    "dimension": "教学班",          // 高一为 "行政班"
    "classes": [
      { "class_label": "物A1", "mine": true,  "main_total_avg": 281.3, "plus3_avg": ..., "total_avg": ..., "source": "computed" },
      { "class_label": "物A2", "mine": false, ... , "source": "class_average" },
      { "class_label": "5",    "mine": false, ... }
    ]
  }]
}
```

聚合规则：
1. **官方班均分优先**：`ClassAverage` 里有的（按 `class_label`，空则 `str(class_num)`）直接用，`source=class_average`。
2. **走班/无官方行→现算**：对我的教学班（及任何只有成员、无班均分的班），用成员学号集合在 `TotalScore` 上 `func.avg` 现算，`source=computed`。复用源码已有的「`student_classes` 子查询 + 按总分类型聚合」回退路径，把分组键从 `class_num` 换成 `class_label`/成员集合。
3. **dimension**：grade==1 固定 `行政班`（横轴=数字班）；grade>=2 优先 `教学班`（数据含 `class_label` 或老师已配教学班），否则回退 `行政班`。
4. **mine 标记**：`class_label ∈ my_class_labels(grade)`。前端据此高亮（替换现有 `targetClass === class_num` 单班高亮逻辑）。

> 「全年级所有班」的来源：成绩表带教学班列时，全年级教学班从 `SubjectScore.class_label` distinct 得到；不带时退化为行政班维度（数字班全列）。我的走班即使全年级数据缺它的官方均分，也用成员现算补一根柱子并高亮。

## 5. Excel 解析器扩展（`ingest/excel_parser.py`）

目标：在不破坏现有高一固定列 / 高二三 3+3 schema 的前提下，**识别可选的「教学班/走班班级」列**。

1. 新增 `parse_class_label(value) -> str | None`：去空白；数字也保留为字符串（`"5"`）；空→None。区别于现有只收数字的 `class_num()`。
2. **学生明细表**：在表头探测阶段，除「班级」(数字行政班) 外，再探测列名 ∈ {`教学班`,`走班`,`走班班级`,`教学班级`,`选科班`} 的列；命中则逐行写 `class_label`。未命中 `class_label=None`（行为同现状）。
3. **班均分表**（`parse_excel_grade23_class_averages` / `normalize_grade1_class_averages`）：同理探测教学班列；命中写 `class_average.class_label`，否则 `class_label = str(class_num)`。
4. 解析输出的 `subject_scores` / `class_averages` dict 增加 `class_label` 键；`ingest/router.py` 写库时落到新列。
5. **上传后钩子**（`ingest/router.py` 的 `parse_and_store` 末尾，commit 前后）：调用 `teaching/service.sync_members_after_upload(exam)`：
   - 对 `kind=行政` 班按 `class_num` 重算 `source=class_num` 成员；
   - 对出现的 `class_label` 自动加 `source=parser` 成员到同名教学班（若老师已配该班）。

> 真实样表的教学班列名/位置需在 Phase 1 用样本核对（假设 A2）。探测用「列名关键字匹配」而非写死列号，降风险。
