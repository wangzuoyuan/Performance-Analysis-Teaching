# 更新日志

本项目所有值得注意的变更记录于此。版本遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [1.0.3] - 2026-07-07

### 修复
- **同名仅姓名学生跨教学班串数据**：仅姓名占位学号此前按姓名全局生成（`_anon:姓名`），若两个教学班各有一名同名（实为不同人）的仅姓名学生，会共用同一占位学号——给一个班录缺交会同时出现在另一个班的看板里，「全部班级」人数也会误去重。现改为按教学班隔离（`_anon:<教学班id>:姓名`），从根上杜绝串班。
  - `anon_sid_for(name, teaching_class_id)` 带上教学班；新增 `is_class_scoped_anon` / `name_from_anon_sid` 兼容新旧格式。
  - 启动迁移 `_rekey_anon_members_class_scoped` 幂等地把历史 `_anon:姓名` 迁成按班隔离格式；仅属单个班的占位成员，其花名册/缺交/特殊/成长档案一并改指到新学号。

### 工程
- 新增 GitHub Actions CI：后端 `pytest`、前端 `tsc --noEmit` + `next build`。

## [1.0.2] - 2026-07-07

### 修复
- **仅姓名教学班成员无法录作业缺交**：在「班级配置」里按姓名添加的教学班成员会落成「仅姓名」占位学号（`_anon:姓名`），但作业录入把成员内连接花名册、`members_of` 又刻意排除占位学号，导致这些成员对作业模块完全不可见——录缺交时提示「未匹配到当前教学班学生」，看板显示「0 名有效学生」。
  - 作业录入改以教学班成员为准匹配（左连花名册回落取姓名），占位成员可按姓名匹配；写入缺交前给缺花名册行的学号补建花名册，记录正常进入看板/预警口径。
  - `members_of` / `all_my_member_ids` 新增 `include_anon` 开关，作业范围纳入仅姓名学生（成绩分析仍默认排除，口径不变）。
  - 新建仅姓名成员时同步补建花名册；启动迁移幂等补齐历史占位成员的花名册（升级后重启一次即自动修复现有数据）。
  - 占位成员「后期补真实学号」时，一并迁移其花名册/缺交/特殊/成长档案。

## [1.0.0] - 2026-07-03

首个正式发布版本：任课老师的多教学班成绩与教学质量分析 Web 应用。

### 教学班（核心）
- **多教学班模型**：一位老师配置任意多个教学班；高一＝行政班数字，高二/三支持走班名（`物A1`/`史B3`）。
- **班级配置向导**（`/settings/classes`）：建班、成员维护、四态批量导入（学号/姓名/花名册）、按行政班号一键同步、排序删班、设当前班。
- **总览 + 分班两种仪表盘**，顶栏一键切换。
- **跨学年学情继承**：分班导致学号变更时，以「姓名确认 + 学号对照表」人工建链，让跨学年画像/趋势/档案接续。

### 成绩分析
- **Excel 批量导入**：学生成绩明细表 + 班级均分表；自动识别教学班/走班列，写入 `class_label`。
- **高一 / 高二高三双口径**：高一主三门/五门/九门；高二/三主三门/+3/3+3 与选考等级分。
- 考试详情、可自定义关注段位、段位趋势、排名筛选/频次、进退步排行、班级对比、学生画像。

### 作业 / 档案 / AI / 备份
- **作业跟踪看板**：智能文本录入缺交/请假/迟到，每日趋势/各科占比/排行/连续缺交预警、自动导出 Excel、花名册排除开关、学期配置。
- **缺交 × 成绩相关性**：缺交次数与排名散点 + 各科皮尔逊系数排序。
- **成长/谈话档案**、**本周关注**（合并连续缺交预警、本周激增、最近考试临界/薄弱/偏科、谈话跟进待办）、**家长会一页纸**。
- **数据备份/恢复**（`~/.exam-tracker-backups`，不被初始化清空）。
- **AI 对话助手**：Anthropic / OpenAI 兼容，只读工具查询本地数据后回答。

### 修复
- **作业特殊记录端点崩溃**：`POST /api/homework/special-records` 调用已删除的 `_find_student_id`，任何请假/迟到提交都会 `NameError`；改用 `_resolve_student`，并支持在教学班内按姓名消歧。
- **负面评价被反转**：智能录入里「不认真」「不工整」等负面词因内含正面子串被误判为正面，导致质量预警与评价分布反转；改为负面词优先匹配。

### 部署
- 本地 `run.py` 跨平台一键启动（后端 `:8000` / 前端 `:3000`）。
- Docker / 群晖 NAS 部署：`docker-compose.yml` + `Caddyfile` + `DEPLOY.md`。
