# 04 · 前端改造蓝图

源：`frontend/src/`。Next 14 App Router + shadcn/ui + Recharts。布局由 `Shell.tsx` 接管（新增页不要自带 header/max-w/min-h-screen）。

## 0. 全局基础设施

### 0.1 班级范围上下文 `ClassScopeContext`（新增 `src/lib/class-scope.tsx`）
- 全局 React Context：`{ classes: TeachingClass[], grade, current: tc_id|'all', setCurrent }`。
- 启动拉 `/api/teaching/classes`；`current` 持久化到后端 `PATCH /api/teaching/current` + localStorage。
- 在 `layout.tsx` 包裹（与 ChatDrawer 同级挂载）。所有页面通过 hook `useClassScope()` 取「当前班 / 全部」。

### 0.2 `labels.ts` 升级
```ts
// 旧 formatClassLabel(grade, classNum) 仅数字
export function formatTeachingClass(tc: {grade:number; label:string}): string  // "高二·物A1"
export function formatClassChip(label: string): string                          // 列表/学生旁徽章
```
保留旧 `formatGradeLabel`。班级标签直接用 `label`（已是 `物A1`/`5` 这类成品串）。

### 0.3 班级范围选择器组件 `ClassScopePicker`（新增）
- 放 `Topbar.tsx`（或各分析页 filter 行），下拉：`全部（我教的班）` + 每个教学班（按 `grade` 分组、`sort_order` 排序）。
- 选中即更新 Context；分析页据此重拉数据（带 `teaching_class_id`，`all` 时不传）。

### 0.4 侧栏 `Sidebar.tsx`
- 底部「当前班级」从单班改为：显示 `current` 教学班 label + 「共 N 个班」；点开可切换（或跳「班级配置」）。
- 导航新增 `班级配置` 入口（`/settings/classes`）。`仪表盘` 拆为「总览」即首页，分班视图走 `/dashboard/[classId]` 或首页内切换（见 §2）。

## 1. 班级配置向导 / 页面 `/settings/classes`（新增）

首次使用（无任何教学班）时，仪表盘横幅引导至此。功能：
1. **按年级建班**：选年级 → 系统从 `/api/teaching/candidate-classes` 列出候选（行政班号 + 扫到的教学班标签）→ 勾选/手填生成教学班，可改 label、选学科、标 `kind`。
2. **维护成员**：
   - 高一/行政班：一键「按行政班号同步成员」(`sync-by-class-num`)。
   - 走班：`粘贴学号` / `粘贴姓名清单` / `上传花名册 Excel` → 调 `members/import`，展示三态结果（已匹配 / 同名待选 / 未匹配补学号）。
3. **设当前班**、排序、删班。
- UI：左列班卡片列表 + 右侧成员表（学号/姓名/来源徽章/行政班）。复用 shadcn `Table`/`Dialog`/`Badge`。

## 2. 仪表盘：总览 + 分班（首页 `/` 改造，对应需求 3）

把现单班仪表盘改成两态，由 `ClassScopePicker` 切：

### 2.1 总览态（`current === 'all'`）
- KPI：跨「我教的所有班」聚合——主三门/+3/3+3 班均可改为「我的班加权均分」或「我的班均分区间」；重点关注=所有班并集计数。
- **考试时间线**：不变。
- **重点关注速览**：合并所有班 focus-list，每个学生项**旁标教学班徽章**（用 `student_class_map` 返回的 label）。← 需求3 重点。
- **本周关注卡 / 备份卡**：保留。本周关注的学生项也加班级徽章。
- 新增「我的班级总览」卡：每个教学班一行（人数、最近考试班均、关注数），点击进分班态。

### 2.2 分班态（选中某教学班）
- 所有 KPI、关注名单、段位分布限定该班成员（请求带 `teaching_class_id`）。
- 顶部显示班名 + 「返回总览」。
- 段位分布、band-trend 图限定本班。

> 实现：首页组件读 `useClassScope()`；`current==='all'` 走聚合接口（多班并发或后端聚合端点 `/api/dashboard/overview`），否则带 `teaching_class_id` 走单班。建议**新增后端聚合端点** `GET /api/dashboard/overview` 返回「我的班级总览」卡数据，避免前端 N 次 fetch。

## 3. 班级对比页 `/compare`（对应需求 2，D2）

现状：单班高亮（`targetClass === class_num`），横轴 `${class_num}班`。改造：

1. 拉 `/api/teaching/classes?grade=` 得 `myLabels: Set<string>`（替换 `targetClassByGrade`）。
2. 消费 `/class/compare` 新结构（`classes[].class_label / mine / source`）。横轴 label 用 `class_label`（`物A1`/`5班`）。
3. **高亮多班**：图表 `Cell` 与排名表行的高亮条件从 `row.class_num === targetClass` 改为 `row.mine`（或 `myLabels.has(row.class_label)`）。可高亮多根柱、多行。
4. 排名表「本班」徽章 → 对每个 `mine` 行都显示；可加一列「来源」区分官方均分/现算（现算的标「估算」小字，提示走班无官方均分）。
5. 顶部「本班」区改为列出我教的所有班 chips（点击可滚动/高亮定位）。
6. `dimension` 字段驱动列头文案（行政班 / 教学班）。
7. 导出 CSV / 打印按钮可顺带实现（现为 disabled TODO）。

## 4. 学生检索 `/student`（对应需求 4）

现状：聚合所有考试 students，按姓名/学号本地搜，班级列 `${class_num}班`。改造：

1. **全局 / 分班切换**：顶部加 `ClassScopePicker`。`all`=我教所有班成员并集；选班=仅该班成员。
   - 数据源建议新增后端 `GET /api/students?teaching_class_id=`（返回我范围内学生 + 所属教学班 label + 最新成绩摘要），替代现在「拉所有 exam 详情前端聚合」的低效做法（现状对每场考试都 `fetch /api/exams/{id}`，班级越多越慢）。
2. **班级列**：显示教学班 `label`（非数字班）。`all` 态下尤其重要——每个学生旁标其教学班。
3. 搜索框保留；过滤集合受当前 scope 限制。
4. 概览卡「学生数」随 scope 变化。

## 5. 考试详情 `/exam/[id]`（分班展示，对应需求 4）

- 现页已有 `rank_bands`（按 class_num 分组）、班级均分、学生成绩矩阵。改造：
  1. 顶部加 `ClassScopePicker`：选班后统计区（`stats`）、学生矩阵限定该班成员；`all` 时看全部（但「我的班」用底色区分）。
  2. `rank_bands` 分组键改 `class_label`，并把「我的教学班」置顶/高亮。
  3. 学生成绩矩阵（桌面宽表 `hidden md:block` + 移动卡片 `md:hidden StudentScoreMobileCards`）每行加教学班徽章列；移动端卡片同。
  4. 请求带 `teaching_class_id`（消费 §03 的 `/exams/{id}?teaching_class_id=`）。

## 6. 学生画像 `/student/[id]`

- 头部展示从 `class_num` 改为教学班 `label`（取 `student_class_map`）。
- 「主三门班级排名」口径已在后端改为教学班成员集合，前端文案标注「（教学班内排名）」。
- 跨学年趋势：高一段标行政班、高二/三段标教学班（趋势点 tooltip 带当年班级）。
- 作业卡 / 档案卡不变。

## 7. 移动端与既有约定

- 沿用 `Shell.tsx` 响应式；新组件窄屏用 `w-full sm:w-[..]` + `flex-col sm:flex-row`。
- 颜色用 `brand-*`/`success`/`warning`/`danger` token；Recharts 内写颜色字符串。
- 「我的班」高亮统一用 `brand-500`；走班「估算」均分用虚线/半透明或 `warning` 提示。
- 缺考字段仍显示 `—`。

## 8. 前端测试 / 自检
- `npx tsc --noEmit` 通过（类型随接口改动同步）。
- 关键路径手测：无班→配置向导→建高一行政班(同步成员)→建高二走班(导入花名册)→总览仪表盘见多班+学生班徽章→分班切换→对比页多班高亮→学生检索分班→考试详情分班。
